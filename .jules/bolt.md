## 2025-02-24 - React.memo and CSS transitions on large DOM subtrees during panning
**Learning:** During drag-and-drop or continuous panning interactions (like in `PreviewArea`), CSS transitions on the transform property can cause layout thrashing and visual lag. Additionally, if the dragged element is a large component (like `YearGrid` with hundreds of DOM nodes for days/weeks/months), parent state updates (like zoom or pan position) will trigger an expensive reconciliation of the entire subtree on every frame of the drag if the component isn't memoized.
**Action:** Always verify that components rendered inside interactive, frequently-updating containers (like panning areas) are wrapped in `React.memo` to prevent cascading re-renders. Furthermore, conditionally disable CSS `transition-transform` classes while a user is actively panning to ensure the browser prioritizes immediate style updates over smoothing transitions, maintaining responsiveness.
